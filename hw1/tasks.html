Homework 1: Tasks for sykaf1

Your personal task is to develop a system for menus and their dishes.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2123.hw1.domain.Dish. With the following private fields:

  1. String name
  2. String description
  3. String dietary
  4. int rating
Generate necessary getters and setters. [7 marks]

Create class co2123.hw1.domain.Menu. With the following private fields:

  1. int id
  2. String type
  3. String chef
  4. List<Dish> dishes
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2123.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Menu> menus in class co2123.hw1.Hw1Application. Initialize the list inside class co2123.hw1.Hw1Application to contain one menu with two dishes. [4 marks]

Create controller co2123.hw1.controller.MenuController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/menus" should show view menus/list. [2 marks]
    • Add the list of all menus from co2123.hw1.Hw1Application to the model as attribute menus. [2 marks]
    • Create the corresponding JSP to list the type and chef of each menu followed by a link to URL "/dishes?menu={id of menu}", e.g., "/dishes?menu=3". [4 marks]
  2. Requests for URL "/newMenu" should show view menus/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="menu" and action="/addMenu" and inputs for all fields of class Menu except for dishes. [5 marks]
  3. POST requests for URL "/addMenu" should add a new menu to the list of all menus in co2123.hw1.Hw1Application and redirect to /menus. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2123.hw1.controller.DishController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/dishes" should show view dishes/list. [2 marks]
    • Based on request parameter menu determine the corresponding menu from co2123.hw1.Hw1Application. [2 marks]
    • Add the list of all dishes of that menu to the model as attribute dishes. [2 marks]
    • Create the corresponding JSP for the view to list the name, description, dietary, and rating of each dish. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newDish?menu={id of menu whose dishes are listed}", e.g., "/newDish?menu=3". [4 marks]
  2. Requests for URL "/newDish" should show view dishes/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="dish" and action="/addDish" and inputs for all fields of class Dish. [6 marks]
    • Also add the request parameter menu with the ID of the current menu. [4 marks]
  3. POST requests for URL "/addDish" should add a new dish to the list of dishes of the menu identified by request parameter menu and then redirect to /menus. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Menu and make it available in controller MenuController. [4 marks]

When adding a new menu validate that

  1. The value of field id is unique within the list Hw1Application.menus. [3 marks]
  2. The value of field type is not empty. [2 marks]
  3. The value of field chef is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Dish and make it available in controller DishController. [4 marks]

When adding a new dish validate that

  1. The value of field name is not empty. [2 marks]
  2. The value of field description is not empty. [2 marks]
  3. The value of field dietary is either "Pescatarian", "Gluten Free", or "Allergen Free". [3 marks]
  4. The value of field rating is between (including) 1 and 10. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
